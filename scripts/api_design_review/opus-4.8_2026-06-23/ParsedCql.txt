public final class ParsedCql
Reviewer: opus-4.8  Date: 2026-06-23  (orchestrator-verified against source)

1. OVERALL ASSESSMENT
ParsedCql is the parsed, POOLED/CACHED form of a CQL statement (originalCql / parameterizedCql /
namedParameters / parameterCount), built via the static factory parse(String) and backed by a
KeyedObjectPool. It is the CQL analogue of the core ParsedSql. The design is sound; the one real issue is
that its only collection accessor leaks the live internal map of a shared, pooled instance.

[RE-REVIEW 2026-06-24: The single Medium finding (mutable live-map leak) is now ALREADY-FIXED. The field is
 now `private final ImmutableMap<Integer,String> namedParameters` (ParsedCql.java:167), built via
 ImmutableMap.wrap(localNamedParameters) in the constructor (:309), and namedParameters() now returns that
 ImmutableMap (return type changed to ImmutableMap, :564-565). A pooled/shared instance can no longer be
 corrupted by a caller. No other findings remained open in this report; class is now clean. See per-finding tag.]
