1. OVERALL ASSESSMENT
The blocking per-collection executor is large but well-organized, with a genuinely strong return-type
discipline: a clean four-way split of Optional / Nullable / null (gett) / empty and a List vs DataSet vs
Stream split that reads clearly. The query/insert/update/delete/aggregate families wrap the driver's
MongoCollection faithfully. The actionable items are a dual-meaning update overload, a small
insert-family return inconsistency, and one accessor-name drift vs the reactive twin.

[RE-REVIEW 2026-06-24] Net of the re-review: the source has materially improved since 2026-06-23. The
accessor rename (coll()) is DONE, the insert-family return inconsistency is ALREADY-FIXED (insertOne ->
InsertOneResult, insertMany -> InsertManyResult, bulkInsert -> BulkWriteResult), and the
updateOne/findOneAndUpdate Collection<?> overloads now document their aggregation-pipeline semantics. The
ONLY remaining design-level item is the dual-meaning (Bson,Object) vs (Bson,Collection<?>) overload
family, which is now well-documented but still dispatched on static type (STILL-VALID, downgraded by the
new javadoc). queryForSingleNonNull's NPE-on-null is STILL-VALID but is now exhaustively documented. The
list(Class)/query(Class) "no-filter convenience absent" item NO-LONGER-APPLIES as written: null filter is
now a deliberate IAE (executeQuery:2496), so the symmetry-with-stream() recommendation is moot.

5. MISSING METHODS
- [RE-REVIEW 2026-06-24: NO-LONGER-APPLIES (as written) — the source now DELIBERATELY rejects a null
  filter on list/query/findFirst: executeQuery does N.checkArgNotNull(filter,"filter") (:2496) and the
  list(Bson):935 / query(Bson):1703 javadoc declares "@throws IllegalArgumentException if filter is null".
  This is an intentional design choice (per the change context), so "add no-filter list/query for symmetry
  with stream()" conflicts with the new contract. If whole-collection list/query is wanted, the supported
  path is stream()/stream(Class) (:2035/:2078) collected to a List, or pass an explicit match-all filter
  (new Document()). Downgrade to a documentation/discoverability note, not a missing-method defect.
