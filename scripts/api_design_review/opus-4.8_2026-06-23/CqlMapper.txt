API DESIGN REVIEW — cassandra/CqlMapper.java
public final class CqlMapper
Reviewer: opus-4.8  Date: 2026-06-23  (orchestrator-verified against source)

1. OVERALL ASSESSMENT
CqlMapper is a registry mapping id -> ParsedCql (+ id -> ImmutableMap attrs), the CQL analogue of the core
SqlMapper. String add(...) overloads parse via ParsedCql.parse; ParsedCql add(...) overloads store
pre-parsed instances. Attrs are owned by the mapper (deliberately NOT by ParsedCql) and exposed immutably.
The design is sound; the one real issue is a split contract within the add(...) overload family.

[RE-REVIEW 2026-06-24: The High split-contract finding is now ALREADY-FIXED — the add(...) family was unified.
 ALL four overloads (ParsedCql and String forms) now return void and reject duplicates via a shared
 checkDuplicateId() helper (CqlMapper.java:467,499,524,561; helper :577). add(String,ParsedCql,Map) now does
 N.checkArgNotNull(parsedCql) (:500), so its Low null-check finding is also ALREADY-FIXED (id still deliberately
 unvalidated). get(String)->null and getAttributes two-state contract remain STILL-VALID by-design Lows. The
 clear()/values()/entrySet() MISSING note remains STILL-VALID (none added; cqlIds()/containsId/size/isEmpty are
 the enumeration surface). NEW: several Javadoc examples reference non-existent members (new CqlMapper(String),
 instance loadFrom(...), keySet()) — see NEW ISSUES. See per-finding tags below.]
