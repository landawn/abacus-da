1. OVERALL ASSESSMENT
A clean, faithful typed projection of MongoCollectionExecutor (binds entity type T + collection). Read/
write/aggregate ops all have correct typed return shapes. No high-confidence defects. The one genuinely
actionable gap is the absence of no-filter "whole collection" convenience overloads; the rest are
intentional conventions. NOTE: this mapper's distinct() routes through a $group/$project pipeline (private
distinctPipeline, :3118) specifically because the driver's native distinct throws
BsonInvalidOperationException on scalar fields — the reactive twin now mirrors this too (fixed 2026-06-24;
see its report).

2. NAMING ISSUES
None.

3. RETURN-TYPE AND PARAMETER ISSUES
- findOneAndUpdate/Replace/Delete return nullable T while get/findFirst return Optional<T> |
  MongoCollectionMapper.java:2865-3083 | Low | Intentional split (mirrors driver + gett convention;
  reactive collapses to Mono<T>). Documented; recorded, no change recommended.
- bulkWrite accepts driver WriteModel<? extends Document> (untyped) unlike typed bulkInsert | :2806,2835
  vs :2747,2776 | Low | The one write method not projected through T; acceptable as a raw passthrough,
  flagged so the typed-vs-raw split is a conscious choice.

4. MISPLACED METHODS
None.

5. MISSING METHODS
- No-arg whole-collection list() / stream() / query() overloads absent | MongoCollectionMapper.java (cf.
  executor stream() at MongoCollectionExecutor.java:2041) | Medium | Every list/stream/query overload
  requires a filter; add no-arg all-entity overloads (delegate empty filter). NOTE: the reactive twin
  also lacks them — fix both or neither.
- findFirst(Bson filter, Bson sort) overload absent | :629-731 | Low | Sort-without-projection currently
  needs a null projection arg; symmetric with executor + reactive twin, so low priority.

CROSS-CLASS NOTES

  delegated to native distinct and threw on scalar fields — a real behavioral divergence. [RESOLVED

  collectionExecutor.aggregate). Correction to the original note: the fix belongs in the reactive MAPPER,
