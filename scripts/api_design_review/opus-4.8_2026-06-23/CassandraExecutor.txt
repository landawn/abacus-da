1. OVERALL ASSESSMENT
The v4 concrete is a thin driver-binding over CassandraExecutorBase. Almost all public API is inherited;
the class declares only driver-v4 specifics (session() -> CqlSession, codec registration, the UDTCodec
helper, StatementSettings) plus a handful of override-for-Javadoc terminals. The v3/v4 divergences are
genuine driver differences (v4 merged Cluster into CqlSession and dropped the runtime object Mapper), NOT
accidental API drift. Only two minor items.

2. NAMING ISSUES
- UDTCodec.create(Session,String userType,String,Class) String param named userType | CassandraExecutor.
  java:2078 | Low | The String 3rd param is a UDT *name*, but sibling create(UserDefinedType userType,...)
  at :1941 uses userType for the type object — conceptual collision. Rename the String param to
  userTypeName. (Matches v3 exactly — consistent, so fix both.)
