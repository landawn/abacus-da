1. OVERALL ASSESSMENT
The v3 concrete mirrors the v4 concrete against the older DataStax driver 3.x. Its v3-only members
(cluster(), mapper(Class)) are correct because driver 4.x merged Cluster into CqlSession and dropped the
built-in MappingManager. The class documents itself as legacy/superseded but is missing the actual
@Deprecated annotation. One shared Javadoc overclaim with v4.
[RE-REVIEW 2026-06-24] Both prior items are now RESOLVED in source: @Deprecated annotation present on the
class (:238) and on v3/AsyncCassandraExecutor (:46); execute(Statement) overclaim removed. ONE new v3-only
item surfaced: the UDTCodec.create String UDT-name param is still `userType` (v4 was renamed to
userTypeName) — Low-severity naming inconsistency (see NEW ISSUES).

2. NAMING ISSUES
None beyond the shared userType String-param item noted in the v4 report (UDTCodec.create) — fix both.
[RE-REVIEW 2026-06-24: STILL-VALID and now v3-ONLY — v3/CassandraExecutor.java:2454
`create(final Cluster cluster, final String keySpace, final String userType, final Class<T> javaClazz)`
still names the UDT-NAME String param `userType`, colliding with the UserType-OBJECT param `userType` on
the sibling create(UserType userType,...) at :2321. v4's twin was renamed to userTypeName; v3 was not, so
the "fix both" is now half-done. Rename v3's String param to userTypeName. See NEW ISSUES.]
