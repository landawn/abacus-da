1. OVERALL ASSESSMENT
The v3 concrete mirrors the v4 concrete against the older DataStax driver 3.x. Its v3-only members
(cluster(), mapper(Class)) are correct because driver 4.x merged Cluster into CqlSession and dropped the
built-in MappingManager. The class documents itself as legacy/superseded but is missing the actual
@Deprecated annotation. One shared Javadoc overclaim with v4.

2. NAMING ISSUES
None beyond the shared userType String-param item noted in the v4 report (UDTCodec.create) — fix both.

3. RETURN-TYPE AND PARAMETER ISSUES
- [RESOLVED 2026-06-23 — doc-fixed] execute(Statement) Javadoc overclaimed settings application |
  v3/CassandraExecutor.java:1542-1543 | Medium | Doc said executor StatementSettings are applied before
  execution, but the body is a raw session.execute(statement) (settings are only applied by the
  prepareStatement(String...) helpers). The v4 twin (same method, raw pass-through) neither applies nor
  claims it, confirming raw execution is the intended design. FIXED: removed the false sentence (doc now
  matches the actual, sibling-confirmed behavior). No code change.
  CORRECTION to the original finding: v4 does NOT contain this sentence — it was a v3-only error, not
  "shared doc drift."

  :229) | High | The class Javadoc declared it deprecated in favor of cassandra.CassandraExecutor (v4),
