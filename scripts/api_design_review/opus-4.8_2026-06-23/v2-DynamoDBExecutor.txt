1. OVERALL ASSESSMENT
The v2 DynamoDB executor mirrors the v1 executor against AWS SDK v2 (DynamoDbClient; *Response types). It
is a coherent thin wrapper with the same SDK-mirror return style (bare nullable getItem, not Optional —
intentional, consistent with v1 and HBase). v2-only differences (Response type renames, no
dynamoDBMapper(), enhanced-client model) are by-design SDK differences, NOT drift. Findings are cross-twin
naming/parity and one validation-uniformity item.

  [RE-REVIEW 2026-06-24: STILL-VALID — v2/AsyncDynamoDBExecutor.java has 193 CompletableFuture, 0

- Dead null branch + contradictory Javadoc in query(QueryRequest,Class<?>) | v2/DynamoDBExecutor.java:
  3095-3121 | Medium | The method now calls N.checkArgNotNull(targetClass) at :3097, so the

  v2/AsyncDynamoDBExecutor.java:1795-1796 | Low | Sync query() now rejects null targetClass; the async twin
