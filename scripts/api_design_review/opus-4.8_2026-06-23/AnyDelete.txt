1. OVERALL ASSESSMENT
AnyDelete is a faithful fluent wrapper over HBase `Delete`. No correctness defects in declared methods.
Findings are minor internal-consistency and cross-class parity items shared with the other AnyMutation
subclasses.

2. NAMING ISSUES
None specific to AnyDelete (slice param naming rowOffset/rowLength matches AnyPut; see SUMMARY for the
cross-subclass offset/length divergence).

3. RETURN-TYPE AND PARAMETER ISSUES
- AnyDelete(Object,long,NavigableMap) ctor / of(Object,long,NavigableMap) | AnyDelete.java:219 / 424 |
  Low | Uses helper toRowBytes(rowKey) while every other AnyDelete ctor uses toRowKeyBytes(rowKey)
  (AnyDelete.java:161,179,192,206). Verified: HBaseExecutor.toRowBytes and toRowKeyBytes both delegate to
  toValueBytes, so behavior is identical — purely a cosmetic inconsistency. Unify on toRowKeyBytes.

4. MISPLACED METHODS
None.

  byte[] factories; AnyDelete (like AnyPut) routes byte[] through of(Object). Cosmetic only
