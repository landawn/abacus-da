HBaseExecutor is a comprehensive, well-structured sync facade over the HBase Table API plus entity
mapping (via @ColumnFamily/@Id and the Any* value objects). The get/scan/exists/put/delete/mutateRow/
append/increment/incrementColumnValue families are symmetric and complete, and the surface is mirrored
1:1 by AsyncHBaseExecutor. The choice to return bare T/Result (null/default) rather than Optional is
applied consistently across the whole class and its async twin — recorded as intentional, not flagged.
The actionable items are one genuine internal naming inconsistency, a documented behavioral divergence in
the typed batch-get, and some dead commented-out code.
