1. OVERALL ASSESSMENT
The shared Cassandra executor base is large and healthy. It applies the same strong four-way return
discipline praised in the mongo executor (Optional via get / null via gett / Nullable / empty), uses a
consistent (String cql, Object... parameters) positional convention with single-special-arg
(Map/array/Collection/entity) binding, and avoids the raw Object[] varargs footgun. No correctness,
overlap, or null/empty-contract bugs. Findings are minor: a vestigial type parameter, a @Beta
inconsistency, and two already-deprecated methods.

(both query overloads now Class<?>; return type renamed DataSet->Dataset). The @Beta delegate

NO-LONGER-APPLIES — AsyncCassandraExecutorBase now also carries 20 @Beta on the matching queryForXxx
methods (parity achieved). All deprecation/keep items intact. No new issues.
