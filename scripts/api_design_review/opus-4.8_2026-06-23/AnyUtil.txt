3. RETURN-TYPE AND PARAMETER ISSUES
- array2Props(Object[]) | aws/AnyUtil.java:70,82 | Low | Silently coerces non-String property names via
  N.stringOf, whereas the analogous pair-parsing in MongoDBBase (toBson/toDocument varargs) rejects
  non-String names with IllegalArgumentException. Align the two on one policy. Confirming check: compare
  to MongoDBBase pair-parser null/type handling.
  [RE-REVIEW 2026-06-24: STILL-VALID, and the divergence WIDENED — line 82 still does
   `props.put(N.stringOf(propNameAndValues[i]), ...)`, coercing non-String names. Meanwhile MongoDBBase's
   pair-parsers now EAGERLY reject non-String names with IllegalArgumentException (MongoDBBase.java:610-613
   toDocument, 673-676 toBSONObject, 778-781 toDBObject — added since the original review). So the two
   policies are now explicitly opposite. NOTE: AnyUtil's coercion is documented as intended behavior in its
   Javadoc (aws/AnyUtil.java:39-41, example `{100, "hundred", ...}` -> `{100=hundred}`), so this is a
   deliberate-but-inconsistent design choice, not a bug.]
