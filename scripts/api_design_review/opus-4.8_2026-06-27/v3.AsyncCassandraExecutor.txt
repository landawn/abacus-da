1. OVERALL ASSESSMENT
Structurally identical to the driver-4 facade and at full method parity with it (same
sync(), same two row-mapper stream overloads, same four execute implementations, same four
pass-through overrides). Driver-version differences are appropriate: driver-3 raw Statement/
BatchStatement.Type/com.datastax.driver.core.* types, the execute impls return the driver-3
ResultSet directly (no ResultSets::wrap, correct since driver-3's async future already
yields a ResultSet), and the whole class is @Deprecated pointing at the driver-4
AsyncCassandraExecutor. Return shapes and Optional/Nullable wrapping are identical to the
driver-4 sibling, so there are no driver3<->driver4 inconsistencies in the async surface.
