1. OVERALL ASSESSMENT
A clean, thin ContinuableFuture wrapper that faithfully mirrors the sync executor's return
shapes (Optional, Nullable, primitive OptionalX, driver result types, Dataset/List/Stream
wrapped in futures) and correctly mirrors the @Beta and @Deprecated markings. The sync()<->
async() bridge is sound. The real problem is not divergent shapes but MISSING methods:
several sync public methods have no async counterpart, confirmed by the fact that BOTH the
sync class and the reactivestreams variant expose them while this class does not.

2. NAMING ISSUES — None.

3. RETURN-TYPE AND PARAMETER ISSUES
[1] queryForXxx @Beta split (mirrored from sync) | AsyncMongoCollectionExecutor.java:1080-1420
    (@Beta) vs 1461, 1502, 1545 (no @Beta) | Med | Same inconsistency as the sync executor;
    fix together so the two classes stay aligned. See MongoCollectionExecutor 3[1].

4. MISPLACED METHODS — None.
